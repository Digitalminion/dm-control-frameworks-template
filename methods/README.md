# Methods Folder

> **Navigation:** [🏠 Root](../README.md) › **Methods**

## Overview

This folder contains the methodology, standards, and templates that guide our cybersecurity control documentation system. Whether you're setting up new documentation, looking for templates, or understanding our approach to NIST controls, this is where you'll find the foundational guidance.

## What You'll Find Here

### Core Files
- **[`agent.md`](AGENT.md)** - Complete AI agent guidance and interaction protocols
- **`README.md`** - This folder context guide

### Subfolders
- **[`templates/`](templates/README.md)** - Ready-to-use templates for organizational context documentation
- **[`adoption/`](adoption/README.md)** - Framework adoption strategies and approaches  
- **[`guidelines/`](guidelines/README.md)** - Documentation standards and best practices

## Getting Started

### If You're New to This System
1. Start with this README to understand the folder structure
2. Check `methods/guidelines/` for documentation standards
3. Use `methods/templates/` when you need to create new context documentation

### If You're Looking for Templates
The `methods/templates/` folder contains templates for:
- **Generic Control Templates** (`controls/`): Universal templates for control documentation structure
- **Framework-Specific Templates** (`frameworks/`): Specialized templates for widely adopted frameworks such as NIST 800-53, 800-171, and CSF
  - **NIST 800-53**: Detailed technical control templates with baseline-specific guidance
  - **NIST 800-171**: CUI-focused templates for non-federal organizations and contractors
  - **NIST CSF**: Strategic, outcome-oriented templates with implementation tier guidance
- **Organizational Templates**: Standardized formats for stakeholder, risk, and technology documentation
- **Meta-Analysis Framework**: Comprehensive template evaluation and improvement methodology

### If You're Working on Framework Implementation
The `methods/adoption/` folder contains our organization's approach to:
- Framework interpretation strategies
- Control family implementation
- Risk-based decision making
- Compliance mapping

### If You're Setting Standards
The `methods/guidelines/` folder provides:
- Documentation quality standards
- Best practices for control documentation
- Review and approval processes
- Version control guidelines

## Folder Purpose

This folder serves as the foundation for all documentation methodology, providing:
- **Consistent templates** for organizational context documentation
- **Proven adoption strategies** for framework implementation
- **Quality standards** and best practices for documentation
- **AI agent guidance** for automated improvement

Use this folder when you need methodology, templates, or standards for control documentation work.
