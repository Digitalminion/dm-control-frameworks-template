# Methods Folder

> **Navigation:** [🏠 Root](../README.md) › **Methods**

## Overview

This folder contains the methodology, standards, and templates that guide our NIST documentation system. Whether you're setting up new documentation, looking for templates, or understanding our approach to NIST controls, this is where you'll find the foundational guidance.

## What You'll Find Here

### Core Files
- **[`agent.md`](AGENT.md)** - Complete AI agent guidance and interaction protocols
- **`README.md`** - This folder context guide

### Subfolders
- **[`templates/`](templates/README.md)** - Ready-to-use templates for organizational context documentation
- **[`adoption/`](adoption/README.md)** - Our organization's NIST framework adoption strategies and approaches  
- **[`guidelines/`](guidelines/README.md)** - Documentation standards and best practices

## Getting Started

### If You're New to This System
1. Start with this README to understand the folder structure
2. Check `methods/guidelines/` for documentation standards
3. Use `methods/templates/` when you need to create new context documentation

### If You're Looking for Templates
The `methods/templates/` folder contains templates for:
- Organizational profiles
- Technology stack documentation
- Risk assessments
- Stakeholder information
- Operational procedures

### If You're Working on NIST Implementation
The `methods/adoption/` folder contains our organization's approach to:
- Framework interpretation strategies
- Control family implementation
- Risk-based decision making
- Compliance mapping

### If You're Setting Standards
The `methods/guidelines/` folder provides:
- Documentation quality standards
- Best practices for NIST control documentation
- Review and approval processes
- Version control guidelines

## Folder Purpose

This folder serves as the foundation for all documentation methodology, providing:
- **Consistent templates** for organizational context documentation
- **Proven adoption strategies** for NIST framework implementation
- **Quality standards** and best practices for documentation
- **AI agent guidance** for automated improvement

Use this folder when you need methodology, templates, or standards for NIST control documentation work.
