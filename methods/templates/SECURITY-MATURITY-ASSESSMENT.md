# Security Responsibility Maturity Assessment

## Overview

This assessment helps organizations determine their security responsibility maturity level to select appropriate control documentation templates and implementation guidance. The assessment evaluates four key dimensions of organizational security responsibility structure.

## Assessment Instructions

### How to Complete This Assessment

1. **Answer all questions honestly** based on your organization's current state
2. **Score each dimension** using the provided scoring guide
3. **Calculate your overall maturity level** using the scoring framework
4. **Use results** to select appropriate template styles and implementation approaches

### Scoring Scale

- **1 Point**: Rarely or never true (0-25% of the time)
- **2 Points**: Sometimes true (26-50% of the time)  
- **3 Points**: Often true (51-75% of the time)
- **4 Points**: Almost always or always true (76-100% of the time)

### Template Selection Guidance

Based on your assessment results:

#### Business-Led Organizations
- **Language Style**: Business-friendly, avoid technical jargon
- **Technical Depth**: Basic, focus on business impact and compliance
- **Implementation Approach**: Vendor-managed or consultant-guided
- **Focus Areas**: Compliance, cost considerations, vendor options
- **Primary Templates**: Business-led variants with vendor guidance

#### IT-Led Organizations  
- **Language Style**: Technical operational, IT infrastructure focus
- **Technical Depth**: Intermediate, include configuration details
- **Implementation Approach**: IT-managed with internal teams
- **Focus Areas**: Technical implementation, IT process integration
- **Primary Templates**: IT-led variants with technical procedures

#### Engineering-Led Organizations
- **Language Style**: Engineering workflow, automation focus
- **Technical Depth**: Advanced, include automation and architecture
- **Implementation Approach**: Integrated with DevOps and engineering
- **Focus Areas**: Automation, architecture patterns, scalability
- **Primary Templates**: Engineering-led variants with infrastructure-as-code

#### InfoSec-Led Organizations
- **Language Style**: Security professional, advanced techniques
- **Technical Depth**: Expert, include threat intelligence and advanced methods
- **Implementation Approach**: Security-owned with autonomous authority
- **Focus Areas**: Threat intelligence, advanced techniques, security operations
- **Primary Templates**: InfoSec-led variants with threat-informed content

- Develop advanced security capabilities
