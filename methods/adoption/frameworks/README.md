# NIST Frameworks

> **Navigation:** [🏠 Root](../../../README.md) › [📁 Methods](../../README.md) › [📁 Adoption](../README.md) › **Frameworks**

## Overview

This folder contains framework-specific implementation approaches for the major NIST cybersecurity frameworks supported by this repository. Each framework has its own subdirectory with detailed implementation checklists, guidance, and AI instructions for systematic adoption.

## Available Frameworks

### [NIST Cybersecurity Framework (CSF)](csf/README.md)
- **Purpose**: Voluntary framework for managing and reducing cybersecurity risk
- **Structure**: 5 functions, 23 categories, 108 subcategories
- **Implementation**: Risk-based approach with organizational profiles
- **Focus**: High-level strategic cybersecurity risk management

### [NIST Special Publication 800-53](800-53/README.md)  
- **Purpose**: Security and privacy controls for federal information systems
- **Structure**: 20 control families with detailed controls and enhancements
- **Implementation**: Risk-based approach with organizational tailoring
- **Focus**: Comprehensive security and privacy controls

### [NIST Special Publication 800-171](800-171/README.md)
- **Purpose**: Requirements for protecting controlled unclassified information (CUI)
- **Structure**: 14 control families derived from NIST 800-53
- **Implementation**: Mandatory for organizations handling CUI
- **Focus**: CUI protection in nonfederal systems

## Framework Relationships

These frameworks work together to provide comprehensive cybersecurity coverage:

- **NIST CSF**: Provides the strategic framework and risk management approach
- **NIST 800-53**: Delivers detailed technical and administrative controls
- **NIST 800-171**: Offers focused CUI protection requirements

Organizations can implement multiple frameworks simultaneously, with controls and requirements mapped between frameworks for efficiency.

## Getting Started

### Choose Your Framework(s)
1. **For strategic cybersecurity management**: Start with NIST CSF
2. **For comprehensive federal compliance**: Implement NIST 800-53
3. **For CUI protection requirements**: Use NIST 800-171
4. **For comprehensive coverage**: Combine multiple frameworks

### Implementation Approach
1. **Review framework-specific guidance** in each subdirectory
2. **Use detailed implementation checklists** to track progress
3. **Apply organizational context** from the `context/` folder
4. **Create comprehensive documentation** using templates from `methods/templates/`
5. **Monitor and update** implementation status regularly

## Folder Purpose

This directory serves as the framework implementation hub, providing:
- **Framework-Specific Guidance** for targeted implementation approaches
- **Detailed Implementation Checklists** for systematic progress tracking
- **AI Agent Instructions** for automated documentation creation
- **Cross-Framework Coordination** for comprehensive cybersecurity coverage

Use this folder when you need specific guidance for implementing particular NIST frameworks within your organizational context.
