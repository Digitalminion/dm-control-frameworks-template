# Risk Context

## Overview

This folder contains documentation about risk environment, threat landscape, vulnerabilities, and compliance requirements that drive NIST control selection and implementation priorities.

## What You'll Find Here

### Risk Assessments and Profiles
- Complete risk profiles using the `risk-profile.tpl.md` template
- Risk assessment results and findings
- Risk treatment strategies and decisions
- Risk monitoring and measurement information

### Threat Landscape
- Current threat environment and intelligence
- Industry-specific and targeted threats
- Emerging threats and trends
- Threat actor profiles and capabilities

### Vulnerability Information
- Vulnerability assessment results
- Technical and operational vulnerabilities
- Remediation status and timelines
- Vulnerability management processes

### Compliance Requirements
- Regulatory and legal requirements
- Industry standards and frameworks
- Contractual security obligations
- Audit findings and remediation status

## Getting Started

### If You're Documenting Risk Environment
1. Start with the risk profile template
2. Document current risk assessment results
3. Include threat intelligence and vulnerability information
4. Map compliance requirements and obligations

### If You're Using Risk Context for Control Implementation
1. Review risk assessments to understand priority risks
2. Use threat information to select appropriate protective controls
3. Reference compliance requirements for mandatory implementations
4. Consider risk tolerance when recommending control approaches

## Template Usage

### Available Templates
- **risk-profile.tpl.md** - Comprehensive risk documentation

### Creating Risk Documentation
1. Use the risk profile template from `methods/templates/`
2. Document all relevant threats, vulnerabilities, and compliance requirements
3. Include risk tolerance levels and acceptance criteria
4. Focus on risks that influence control selection and prioritization
5. Save with descriptive names (e.g., `acme-corp-risk-profile.md`)

## Documentation Standards

### Risk Documentation Requirements
- All significant risks should be documented and assessed
- Include current threat intelligence and trends
- Document compliance requirements comprehensively
- Map risks to potential control implementations
- Include risk treatment decisions and rationale

### Update and Maintenance
- Risk documentation should be updated regularly
- Threat intelligence should be current and relevant
- Vulnerability information should reflect current status
- Compliance requirements should be monitored for changes

## Folder Purpose

This folder serves as the risk knowledge repository, providing:
- **Risk Context** for understanding what needs to be protected
- **Threat Information** for selecting appropriate defensive controls
- **Compliance Requirements** for mandatory control implementations
- **Prioritization Guidance** for focusing implementation efforts

Use this folder when you need to understand the risk environment for control prioritization and implementation planning.
