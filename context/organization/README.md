# Organization Context

> **Navigation:** [🏠 Root](../../README.md) › [📁 Context](../README.md) › **Organization**

## Overview

This folder contains documentation about organizational structure, governance, stakeholders, and operational characteristics that influence control implementation approaches and success factors.

## What You'll Find Here

### Organizational Profiles
- Complete organizational profile documentation using the `organization-profile.tpl.md` template
- Organizational structure and reporting relationships
- Business objectives and mission alignment
- Industry context and regulatory environment

### Stakeholder Documentation
- Stakeholder roles and responsibilities using the `stakeholders.tpl.md` template
- Decision-making structures and approval processes
- Communication preferences and escalation procedures
- Training and awareness responsibilities

### Operational Constraints
- Resource limitations using the `constraints.tpl.md` template
- Budget and financial constraints
- Staffing and skill limitations
- Timeline and scheduling constraints

### Governance and Culture
- IT governance model and decision-making processes
- Organizational culture and change management capabilities
- Risk tolerance and acceptance criteria
- Compliance requirements and audit processes

## Getting Started

### If You're Documenting a New Organization
1. Start with the organizational profile template
2. Document all key stakeholders and their roles
3. Identify and document operational constraints
4. Gather governance and cultural information

### If You're Using Organization Context for Implementation
1. Review the organizational profile for structure and governance
2. Identify key stakeholders for specific control areas
3. Consider constraints when planning implementation approaches
4. Align recommendations with organizational culture and processes

## Template Usage

### Available Templates
- **organization-profile.tpl.md** - Complete organizational profiling
- **stakeholders.tpl.md** - Stakeholder mapping and roles
- **constraints.tpl.md** - Resource and operational limitations

### Creating Organization Documentation
1. Use the appropriate template from `methods/templates/`
2. Fill in all sections with specific organizational information
3. Update the AI header with organization-specific metadata
4. Save files with descriptive names (e.g., `acme-corp-profile.md`)

## Folder Purpose

This folder serves as the organizational knowledge repository, providing:
- **Structural Context** for understanding how the organization operates
- **Stakeholder Information** for effective implementation coordination
- **Constraint Documentation** for realistic implementation planning
- **Cultural Context** for change management and adoption strategies

Use this folder when you need to understand the organizational environment for control implementation planning and execution.
