# NIST CSF Template Quality Checklist

> **Navigation:** [🏠 Root](../../README.md) › [📁 Development](../README.md) › [📋 Checklists](README.md) › **NIST CSF Template Checklist**

## Executive Summary

This comprehensive checklist ensures all NIST CSF templates meet the high-quality standards established by the best templates in the repository. It provides systematic validation criteria for template creation, enhancement, and quality assurance based on analysis of high-quality templates like GV.OC-01 (711 lines) and ID.AM-1 (466 lines).

### Quality Standards Overview

**✅ High-Quality Template Characteristics:**
- **Comprehensive AI Headers**: 85-120 lines with detailed instructions and maturity adaptations
- **Framework Fidelity**: Exact NIST language preservation with strategic value articulation
- **Implementation Depth**: Tier-specific guidance with practical roadmaps
- **Organizational Context**: Comprehensive integration with business and technology context
- **Cross-Reference Integration**: Related subcategory and framework relationships
- **Quality Validation**: Built-in assessment and measurement frameworks

**⚠️ Common Quality Issues:**
- **Minimal Templates**: 20-line placeholders lacking implementation guidance
- **Missing Context**: Insufficient organizational and technology integration
- **Incomplete Sections**: Missing assessment, measurement, or improvement frameworks
- **Poor Cross-References**: Limited relationship mapping between subcategories

## AI Header Quality Checklist

### ✅ AI Header Structure (Lines 1-85/120)

**Required Elements:**
- [ ] **BEGIN AI HEADER**: Accurate line count (count actual lines to END AI HEADER)
- [ ] **AI FILE SUMMARY**: Complete metadata with all required fields
  - [ ] TOPICS: Specific to subcategory and function
  - [ ] CONTENT: Template type and guidance scope
  - [ ] RELATED: Relevant template and context folders
  - [ ] RATING: Appropriate quality rating (foundational, intermediate, advanced)
  - [ ] PURPOSE: Clear template purpose and scope
  - [ ] UPDATE: Update frequency (High, Medium, Low)
  - [ ] Updated: Current date (YYYY-MM-DD format)
  - [ ] Optional: SECURITY_MATURITY_LEVEL for advanced templates
  - [ ] Optional: TARGET_AUDIENCE for maturity-aware templates

**AI COMMENT Section:**
- [ ] **Template Purpose**: Clear explanation of subcategory focus and outcomes
- [ ] **CSF Specificity**: Framework-specific language and intent preservation
- [ ] **Template Usage**: Clear guidance for agent usage and customization

**AI INSTRUCTION Section:**
- [ ] **Standard Operations**: 10+ specific instructions for template usage
- [ ] **Maturity Adaptations**: Where applicable, include maturity-specific guidance
- [ ] **Context Requirements**: Clear guidance for organizational context integration
- [ ] **Quality Validation**: Specific quality criteria and validation requirements

### ✅ Header Quality Validation

**Line Count Accuracy:**
- [ ] BEGIN AI HEADER line count matches actual header length
- [ ] END AI HEADER line count matches actual header length
- [ ] Header includes all required sections with proper formatting

**Metadata Completeness:**
- [ ] All required metadata fields are present and accurate
- [ ] Topics reflect specific subcategory and function
- [ ] Related files are correctly referenced
- [ ] Purpose statement is clear and specific

## Content Structure Quality Checklist

### ✅ Subcategory Information Section

**Required Elements:**
- [ ] **Subcategory ID**: Exact NIST CSF identifier (e.g., GV.OC-01)

## Quality Standards Summary

### ✅ Minimum Quality Threshold

**For Basic Templates:**
- [ ] 200+ lines of comprehensive content
- [ ] Complete AI header with accurate line counting
- [ ] All required sections present and complete
- [ ] Framework fidelity maintained
- [ ] Implementation guidance provided for all tiers
- [ ] Assessment and measurement frameworks included

**For Comprehensive Templates:**
- [ ] 400+ lines of comprehensive content
- [ ] Advanced AI header with detailed instructions
- [ ] Maturity-aware adaptations where applicable
- [ ] Comprehensive cross-reference integration
- [ ] Advanced implementation guidance
- [ ] Sophisticated assessment and measurement frameworks

### ✅ Quality Validation Process

**Before Finalizing Template:**
1. **Run Complete Checklist**: Validate against all checklist items
2. **Verify Line Counts**: Ensure accurate AI header line counting
3. **Check Framework Fidelity**: Validate NIST language and intent preservation
4. **Validate Implementation Guidance**: Ensure comprehensive tier-specific guidance
5. **Verify Cross-References**: Confirm related subcategory relationships
6. **Test Placeholder Replacement**: Ensure all placeholders have guidance
7. **Validate Organizational Context**: Confirm business and technology integration
8. **Check Quality Standards**: Ensure minimum quality thresholds met

**Quality Assurance Checklist:**
- [ ] Template meets minimum quality thresholds
- [ ] All required sections are complete and comprehensive
- [ ] Framework fidelity is maintained throughout
- [ ] Implementation guidance is practical and scalable
- [ ] Cross-reference integration is comprehensive
- [ ] Assessment and measurement frameworks are included
- [ ] Organizational context integration is complete
- [ ] Template is ready for agent use and customization
